* 概要

このドキュメントは =README.org= の Emacs Lisp コードブロックから抽出した詳細設計書です。
全 81 モジュールの責務・公開 API・フック登録・設定変数をレイヤーごとに記述します。

[[file:emacs_config_layer_architecture.svg]]

** 設計原則

| 原則 | 内容 |
|------|------|
| 決定論的起動 | =modules.el= の明示的ロード順、=condition-case= によるエラー隔離 |
| レイヤー分離 | 上位→下位方向のみ依存、依存逆転禁止 |
| 削除安全性 | =my:modules-skip= によるスキップ、=fboundp= ガード |
| Literate Programming | =README.org= → =org-babel-tangle= → =lisp/<layer>/<module>.el= |
| 前方互換性 | Emacs 30+ 対象、31+ 想定、=if-let*= / =when-let*= 使用 |
| 命名規則 | 関数 =my/= or =<module>-= 、パス変数 =my:d:*= 、カスタム =<group>-= |

** ブート順序

#+begin_example
early-init.el
  → init.el (straight.el / leaf / org ブートストラップ)
    → modules.el (my/modules-load)
      → core-* → ui-* → auth-* → completion-* → orgx-* → vcs-* → dev-* → utils-*
        → after-init-hook (core-switches: UI/LSP バンドル有効化)
#+end_example

** ディレクトリ構造

#+begin_example
.emacs.d/
├── early-init.el         ; GC・ディレクトリ・UI 早期設定
├── init.el               ; straight.el・leaf・module 起動
├── modules.el            ; ロード順定義・エラー隔離
├── README.org            ; Literate source (唯一の真実)
├── lisp/core/            ; ランタイム基盤 (17 modules)
├── lisp/ui/              ; 視覚・操作 (16 modules)
├── lisp/auth/            ; 認証 (1 module)
├── lisp/completion/      ; 補完 (12 modules)
├── lisp/orgx/            ; Org 拡張 (8 modules)
├── lisp/vcs/             ; バージョン管理 (3 modules)
├── lisp/dev/             ; 開発ツール (15 modules)
├── lisp/utils/           ; 汎用ヘルパー (9 modules)
├── personal/             ; ユーザー固有設定 (最後にロード)
├── .var/                 ; ランタイム状態 (no-littering)
├── .etc/                 ; 設定ファイル (no-littering)
└── .cache/               ; 一時キャッシュ・ELN
#+end_example

*** 責務

- =package.el= 無効化 (=package-enable-at-startup = nil=)
- GC・ファイルハンドラ起動時最適化 → =emacs-startup-hook= で復元
- ディレクトリ定義: =my:d=, =my:d:var=, =my:d:etc=, =my:d:cache=, =my:d:eln-cache=, =my:d:treesit=
- no-littering 互換変数 (=no-littering-var-directory= 等) の事前定義
- macOS Homebrew ツールチェーン環境変数設定
- =straight.el= ベースディレクトリ・プロファイル設定
- ネイティブコンパイル ELN パス設定
- 早期 UI デフォルト (tool-bar/menu-bar/scroll-bar 非表示、フルスクリーン)
- auto-save / backup を初期段階で無効化（core-editing.el でも明示的に設定）

** core/core-native.el

:PROPERTIES:
:CUSTOM_ID: core-native
:END:

*** 責務

- defines `core-native-comp-report-mode` (silent | collect | verbose)
- collects native-comp warnings up to `core-native-comp-collect-limit`
- emits a notice when the count exceeds `core-native-comp-warn-threshold`
- forces verbose output in CI when `core-native-comp-ci-force-verbose` is set
- optionally auto-saves collected warnings to a file on exit

*** ロード順序ノート

Loaded last in the core/ section (after core-api) so that all core
infrastructure is available.  Native compilation is asynchronous and
initialisation is deferred to after-init-hook, so late load order is safe.

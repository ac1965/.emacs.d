** 設計原則
:PROPERTIES:
:CUSTOM_ID: design-principles
:END:

| 原則 | 内容 |
|------|------|
| *決定論的起動* | =modules.el= の明示的ロード順、=condition-case= によるモジュール単位エラー隔離。起動失敗が上位レイヤーに波及しない。 |
| *レイヤー分離* | 依存は下位→上位方向のみ。core → ui → completion → orgx → dev の順。依存逆転禁止。 |
| *削除安全性* | =my:modules-skip= でモジュールをスキップ可能。=fboundp= / =boundp= ガードにより欠損モジュールで Emacs が起動不能にならない。 |
| *Literate Programming* | =README.org= が唯一の真実。=org-babel-tangle= → =lisp/<layer>/<module>.el=。手動編集不可。 |
| *前方互換性* | Emacs 30+ 対象・31+ 想定。=if-let*= / =when-let*= / =and-let*= 使用。obsolete API 禁止。 |
| *setopt 統一* | すべての =defcustom= 変数は =setopt= で設定。=leaf :custom= / =:pre-setq= を優先。=:config= 内でも原則 =setopt=。ただし backquote 展開・関数参照を含む runtime 評価値（=org-capture-templates= 等）は =setopt= の型チェッカーが誤検知するため =setq= を使用する。 |
| *命名規則* | 公開関数: =my/= or =<module>-=。内部ヘルパー: =<module>--= (double dash)。混在形 =my/<topic>--= は禁止。パス変数: =my:d:*=。 |
| *ドキュメント* | 公開 defun はすべてドキュメント文字列を持つ。=defcustom= を持つモジュールは =defgroup= を宣言し、=:group= に自モジュールのグループを指定。 |

** コーディングルール
:PROPERTIES:
:CUSTOM_ID: coding-rules
:END:

| ルール | 詳細 |
|--------|------|
| =lexical-binding: t= | 全モジュール必須 |
| =(provide 'FEATURE)= | タングルファイル名と一致すること |
| =Built-in は :straight nil= | Emacs 組み込みパッケージは =:straight nil= を宣言 |
| =leaf= 順序 | =:straight= → =:bind= → =:hook= → =:custom= → =:config= |
| =setopt for defcustom= | 全 =defcustom= 変数は原則 =setopt= / =leaf :custom= で設定。=setq= は =defvar=・内部状態・一時オーバーライドのみ。例外: backquote 展開や関数参照を含む runtime 評価値は =setopt= の型チェッカーが誤検知するため =setq= を使用する（=org-capture-templates= 等） |
| 公開 API のみ使用 | サードパーティの =--= プレフィックスシンボルは =fboundp= ガード下でのみ使用 |
| named defun | 無名 lambda の =add-hook= 禁止。named defun を使うこと |
| docstring 必須 | 公開 defun は全てドキュメント文字列を持つ |
| =defgroup= 宣言 | =defcustom= を持つモジュールは =defgroup= を宣言 |

*** 責務

Minimal and version-scoped compatibility fixes for Emacs. Load-order note: core-fixes is the FIRST module loaded (see modules.el).  core-tools loads at position 9.  Therefore this file must NOT defalias to core-tools-emacs>= — that symbol is not yet defined at load time and would raise void-function. core-fixes--emacs>= is defined independently here.  core-tools.el defines its own core-tools-emacs>= separately.  Both are identical one-liners, and the duplication is deliberate: the two functions serve different namespaces (internal vs public) and avoiding the duplication via defalias requires a load-order inversion that would break the deterministic module sequence.

Persistent *scratch* buffer: recreate it automatically when killed. This file: - watches `kill-buffer-hook` to detect *scratch* being killed - schedules recreation via `run-with-idle-timer 0` so the kill fully completes before the new buffer is created - initialises the replacement buffer in `lisp-interaction-mode` Design notes: - `defun` forms are at top level, not inside `leaf` blocks.

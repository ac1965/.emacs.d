** Design Principles
:PROPERTIES:
:CUSTOM_ID: design-principles
:END:

| Principle | Description |
|-----------|-------------|
| *Deterministic startup* | =modules.el= defines an explicit load order; =condition-case= provides per-module error isolation so one failure cannot cascade to higher layers. |
| *Layer separation* | Dependencies flow downward only: core → ui → completion → orgx → dev.  Dependency inversion is prohibited. |
| *Delete safety* | Any module can be skipped via =my:modules-skip=.  =fboundp= / =boundp= guards prevent a missing module from making Emacs unbootable. |
| *Literate programming* | =README.org= is the single source of truth.  =org-babel-tangle= generates =lisp/<layer>/<module>.el=.  The generated files must never be edited manually. |
| *Forward compatibility* | Targets Emacs 30+, anticipates Emacs 31+.  Uses =if-let*= / =when-let*= / =and-let*=.  Obsolete APIs are prohibited. |
| *setopt uniformity* | All =defcustom= variables are set with =setopt= or =leaf :custom=.  =setq= is reserved for =defvar=, internal mutable state, and runtime-evaluated values whose backquote expansion would trigger =setopt= type-check false positives (e.g. =org-capture-templates=). |
| *Naming conventions* | Public functions: =my/= or =<module>-=.  Private helpers: =<module>--= (double dash).  Mixed form =my/<topic>--= is prohibited.  Path variables: =my:d:*=. |
| *Documentation* | All public =defun= must carry a docstring.  Every module owning =defcustom= variables must declare a =defgroup= and reference it in each variable's =:group=. |

** Coding Rules
:PROPERTIES:
:CUSTOM_ID: coding-rules
:END:

| Rule | Detail |
|------|--------|
| =lexical-binding: t= | Required in every module. |
| =(provide 'FEATURE)= | Symbol must match the tangle file name without the =.el= extension. |
| Built-ins: =:straight nil= | Emacs built-in packages must declare =:straight nil= in their =leaf= form. |
| =leaf= keyword order | =:straight= → =:ensure= → =:after= → =:require= → =:pre-setq= → =:custom= → =:bind= → =:hook= → =:init= → =:config= |
| =setopt= for =defcustom= | All =defcustom= variables use =setopt= or =leaf :custom=.  =setq= is used only for =defvar= and internal state.  Exception: backquote expressions with runtime evaluation use =setq= (=org-capture-templates=, =org-agenda-files=, =org-todo-keywords=, =org-refile-targets=). |
| Public API only | Third-party =--= symbols may only be called under an =fboundp= guard. |
| Named =defun= | Anonymous lambdas in =add-hook= are prohibited.  Use named =defun= at module top level. |
| Docstrings required | All public =defun= must carry a docstring. |
| =defgroup= required | Every module that owns =defcustom= variables must declare =defgroup=. |
| =keymap-set= API | Use =keymap-set= / =keymap-global-set= (Emacs 29+).  =define-key= + =kbd= is prohibited. |
| =defun= placement | All =defun= declarations at module top level.  Never inside =leaf= blocks or =with-eval-after-load=. |

** Boot Sequence
:PROPERTIES:
:CUSTOM_ID: boot-sequence
:END:

#+begin_example
early-init.el
  → init.el (straight.el / leaf bootstrap)
      ① (defvar my:modules-extra nil)     ; declared before personal overlay loads
      ② personal/user.el loaded           ; appends to my:modules-extra
      ③ (require 'modules nil t)
      ④ my/modules-load                   ; condition-case isolated loading
          core-* (16) → ui-* (13+α) → auth-* (1) → completion-* (12)
          → orgx-* (9) → vcs-* (3) → dev-* (13) → utils-* (10)
          → additional modules in my:modules-extra
      ⑤ after-init-hook                   ; core-switches: LSP/UI bundle activation
#+end_example

** early-init.el
:PROPERTIES:
:CUSTOM_ID: early-init
:END:

| Item | Value |
|------|-------|
| Tangle target | =early-init.el= |
| provide | — (not a loadable feature) |

*** Responsibilities

Runs before =package.el= initialisation and before the frame is rendered.
Establishes infrastructure only; loads no features.

- Disable =package.el= auto-initialisation
- Set =straight-base-dir= and =straight-build-dir= (versioned: =build-31.0=)
- Define directory constants =my:d:var=, =my:d:cache=, =my:d:etc=
- Temporarily relax GC threshold and clear =file-name-handler-alist=
- Prepare =native-comp-eln-load-path= and tree-sitter grammar path
- Apply early UI defaults (disable toolbar, scroll bar) to prevent flicker

*** Public API

- =my/ensure-directory-exists= — creates a directory path if absent (available from startup)

*** Notes

=straight-build-dir= must be set here, before straight.el bootstraps.
=file-name-handler-alist= is saved to =core--orig-file-name-handler-alist= and restored
in =after-init-hook=; failure to restore breaks TRAMP.

** init.el
:PROPERTIES:
:CUSTOM_ID: init-el
:END:

| Item | Value |
|------|-------|
| Tangle target | =init.el= |
| provide | — |

*** Responsibilities

Activates feature logic after infrastructure is established.

- Configure URL state paths before =url.el= loads
- Bootstrap straight.el
- Import macOS login-shell environment (=exec-path-from-shell=, GUI and daemon)
- Initialise =leaf=
- Apply runtime performance settings (=gcmh=, IO buffers)
- Declare =my:modules-extra= before personal overlay loads
- Load =personal/<login-name>.el=
- Require =modules.el=
- Report startup time and GC count on =after-init-hook=

*** Public API

None.  =bootstrap--safe-load-file= is an internal helper.

*** Notes

=(bound-and-true-p straight-base-dir)= respects the value set in =early-init.el=.
=my:modules-extra= must be declared before personal overlay loads so the personal
overlay can append to it.

** modules.el
:PROPERTIES:
:CUSTOM_ID: modules-el
:END:

| Item | Value |
|------|-------|
| Tangle target | =lisp/modules.el= |
| provide | =modules= |

*** Responsibilities

Central orchestration layer.  Contains orchestration only; feature logic
resides inside the individual module files.

- Define the canonical module load order (=my:modules= list)
- Provide =my:modules-skip= and =my:modules-extra= as =defcustom= (customisable via =personal/=)
- Load each module with =condition-case= error isolation
- Report per-module load time when =my:modules-verbose= is non-nil
- Prevent dependency inversion between layers

*** Public API

- =my/modules-load= — execute the isolated load sequence

*** defcustom Variables

- =my:modules-skip= — list of module symbols to skip
- =my:modules-extra= — additional modules appended after the canonical list
- =my:modules-verbose= — non-nil to enable per-module timing output

*** Notes

=condition-case= catches =error= only; =quit= (=C-g=) propagates so that user
interruption is respected.

* Appendix B: Naming Convention Reference
:PROPERTIES:
:CUSTOM_ID: appendix-b
:END:

| Pattern | Example | Use |
|---------|---------|-----|
| =my/<n>= | =my/safe-load-file= | Public interactive commands and cross-module public functions |
| =<module>-<n>= | =core-tools-emacs>=, =ui-font-apply= | Module public API (prefixed with module name) |
| =<module>--<n>= | =core-custom--ensure-file=, =modules--require-safe= | Private module helpers (double dash = private).  Form =my/<topic>--= is prohibited. |

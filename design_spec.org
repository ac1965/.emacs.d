** 設計原則
:PROPERTIES:
:CUSTOM_ID: design-principles
:END:

| 原則 | 内容 |
|------|------|
| *決定論的起動* | =modules.el= の明示的ロード順、=condition-case= によるモジュール単位エラー隔離。起動失敗が上位レイヤーに波及しない。ロード順 #1 のモジュールは #9 以降の公開 API に依存できない。同一ロジックの重複が生じてもロード順の逆転より優先する。 |
| *レイヤー分離* | 依存は下位→上位方向のみ。core → ui → completion → orgx → dev の順。依存逆転禁止。 |
| *削除安全性* | =my:modules-skip= でモジュールをスキップ可能。=fboundp= / =boundp= ガードにより欠損モジュールで Emacs が起動不能にならない。 |
| *Literate Programming* | =README.org= が唯一の真実。=org-babel-tangle= → =lisp/<layer>/<module>.el=。手動編集不可。 |
| *前方互換性* | Emacs 30+ 対象・31+ 想定。=if-let*= / =when-let*= / =and-let*= 使用。obsolete API 禁止。 |
| *setopt 統一* | すべての =defcustom= 変数は =setopt= で設定。=leaf :custom= / =:pre-setq= を優先。=:config= 内でも =setq= ではなく =setopt=。 |
| *命名規則* | 公開関数: =my/= or =<module>-=。内部ヘルパー: =<module>--=（double dash）。混在形 =my/<topic>--= は禁止。パス変数: =my:d:*=。 |
| *ドキュメント* | 公開 defun はすべてドキュメント文字列を持つ。=defcustom= を持つモジュールは =defgroup= を宣言し、=:group= に自モジュールのグループを指定。 |

** コーディングルール
:PROPERTIES:
:CUSTOM_ID: coding-rules
:END:

| ルール | 詳細 |
|--------|------|
| =lexical-binding: t= | 全モジュール必須 |
| =(provide 'FEATURE)= | タングルファイル名と一致すること |
| =Built-in は :straight nil= | Emacs 組み込みパッケージは =:straight nil= を宣言 |
| =leaf= 順序 | =:straight= → =:bind= → =:hook= → =:custom= → =:config= |
| =setopt for defcustom= | 全 =defcustom= 変数は =setopt= / =leaf :custom= で設定。=setq= は =defvar=・内部状態・一時オーバーライドのみ |
| 公開 API のみ使用 | サードパーティの =--= プレフィックスシンボルは =fboundp= ガード下でのみ使用 |
| named defun | 無名 lambda の =add-hook= 禁止。named defun を使うこと |
| docstring 必須 | 公開 defun は全てドキュメント文字列を持つ |
| =defgroup= 宣言 | =defcustom= を持つモジュールは =defgroup= を宣言 |

** ブート順序
:PROPERTIES:
:CUSTOM_ID: boot-sequence
:END:

#+begin_example
early-init.el
  → init.el (straight.el / leaf / org ブートストラップ)
      ① (defvar my:modules-extra nil)     ← personal ロードより先に宣言 (Fix B)
      ② personal/user.el をロード         (my:modules-extra に ui-visual-aids 等を追加)
      ③ (require 'modules nil t)           defcustom my:modules-extra は no-op
      ④ my/modules-load                    condition-case で隔離ロード
          core-* (16) → ui-* (11+α) → auth-* (1) → completion-* (12)
          → orgx-* (7) → vcs-* (3) → dev-* (13) → utils-* (9)
          → my:modules-extra の追加モジュール
      ⑤ after-init-hook                    core-switches: UI/LSP バンドル有効化
#+end_example

=my:modules-extra= の =defcustom= 実体は =modules.el= に定義されているが、init.el での =defvar= 先行宣言により no-op となる。これは personal ロードの前に変数を確実に存在させるための意図的設計である。

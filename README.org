** Eary-init.el
Key highlights:
- **Garbage Collection Tuning**: You've set up a high initial threshold for garbage collection to avoid frequent pauses during startup, with a reset mechanism after initialization.
- **Native Compilation**: Your configuration ensures proper handling of native compilation caches and suppresses warnings.
- **File Handling**: Temporarily disabling `file-name-handler-alist` during startup improves loading speed.
- **GUI and Startup Settings**: You’ve configured a minimalist and efficient GUI setup, particularly with elements like frame resizing, menu bar handling, and fullscreen mode.

** init.el
### Highlights:
- **Startup Optimization**:
   - By setting `gc-cons-threshold` to a high value, it minimizes the impact of garbage collection during startup.
   - You’re ensuring that Emacs always prefers newer byte-compiled files by setting `load-prefer-newer` to `t`.

- **Org-mode-based Configuration**:
   - Using `org-babel-load-file` to load the main configuration from an `org-mode` file is an efficient and well-documented approach. This makes your configuration cleaner and easier to maintain.

- **Error Handling**: You’ve disabled `debug-on-error`, which can be useful for non-development environments, but may be worth re-enabling if you are debugging your config.

** Personal Configuration aka. tjy.el
Here's a summary of what it does:

### 1. **Personal Information & General Config**
   - Sets your full name, email, preferred font (`HackGen35`), and font size.
   - Defines important directories such as cloud storage and blog paths.
   - Adds custom elisp directories to `load-path`.
   - Sets up font configurations for Emacs when running as a daemon or in window mode.

### 2. **MEW Email Client**
   - Configures MEW as the email client within Emacs, with support for sending and composing emails.

### 3. **Org-mode Setup**
   - Configures Org-mode with custom keybindings for agenda (`C-c a`), capture (`C-c c`), and showing specific org files (`gtd.org`, `notes.org`, `kb.org`).
   - Integrates various org-mode extensions such as:
     - `org-superstar` for pretty bullets.
     - `org-journal` for daily entries.
     - `org-cliplink` for capturing web links into org files.
     - `org-download` for adding images.
     - `org-web-tools` for capturing web content.
     - `toc-org` for generating tables of contents in org and markdown files.
   - Enables `ox-hugo` for Hugo static site generation and sets up capture templates for blog posts.

### 4. **Org-roam Setup**
   - Sets up Org-roam for personal knowledge management and daily notes.
   - Configures keybindings for Org-roam-related tasks like inserting and capturing nodes.

### 5. **ChatGPT, CodeGPT, DALL-E Integration**
   - Prepares Emacs for ChatGPT, CodeGPT, and DALL-E integration using the `openai` packages.

### 6. **Miscellaneous Functions**
   - Adds a function to generate and insert a YouTube iframe link in org files.
   - Integrates Visual Studio Code for opening files in VSCode (`C-c C-v`).
   - Provides a function to display Emacs build information.
   - Adds a function to convert an Org-mode task to a reminder (using macOS Reminders).

    (defun my/toggle-window-split ()
      "Toggle window split between horizontal and vertical."
      (interactive)
      (if (= (count-windows) 2)
          (let* ((this-win-buffer (window-buffer))
                 (next-win-buffer (window-buffer (next-window)))
                 (this-win-edges (window-edges (selected-window)))
                 (next-win-edges (window-edges (next-window)))
                 (this-win-2nd
                  (not (and (<= (car this-win-edges)
                                (car next-win-edges))
                            (<= (cadr this-win-edges)
                                (cadr next-win-edges)))))
                 (splitter
                  (if (= (car this-win-edges)
                         (car (window-edges (next-window))))
                      'split-window-horizontally
                    'split-window-vertically)))
            (delete-other-windows)
            (let ((first-win (selected-window)))
              (funcall splitter)
              (if this-win-2nd (other-window 1))
              (set-window-buffer (selected-window) this-win-buffer)
              (set-window-buffer (next-window) next-win-buffer)
              (select-window first-win)
              (if this-win-2nd (other-window 1))))))

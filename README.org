* Introduction
:PROPERTIES:
:CUSTOM_ID: introduction
:END:

A modern, literate Emacs configuration written in Org Babel. It focuses on fast startup, native compilation, a clean completion UX, first-class LSP support (Eglot or lsp-mode), AI-assisted workflows, and a pragmatic Org/Magit toolchain.

** Features
:PROPERTIES:
:CUSTOM_ID: features
:END:

- *Performance & Native Compilation* — Early init relocates ELN under =.cache/=, suppresses async warnings, widens GC during startup (then restores), and enables *GCMH* for idle GC behavior.
- *Language Server Protocol* — Backend-agnostic helpers with a single switch to choose *Eglot* or *lsp-mode*. Servers auto-enable only when they’re detectable.
- *AI Integration* — *Aidermacs* (vterm backend). Prefers *OpenRouter* when =OPENROUTER_API_KEY= is present; otherwise uses *OpenAI* via =OPENAI_API_KEY=.
- *Modern UI & Editing* — Tree-sitter remaps (=*-ts-mode=), *ef-themes* + *spacious-padding*, Nerd Icons, *Vertico/Orderless/Corfu/CAPE/Embark/Consult*, and switchable *Doom*/*Nano* modelines.
- *Productivity Stack* — Opinionated Org suite (agenda, capture, journal, roam, download, TOC), *Magit* + *diff-hl/Forge*, REST client, Docker/dev helpers, tidy backups, and =auto-save-visited=.

** Coding Rules
:PROPERTIES:
:CUSTOM_ID: conventions
:END:

- =lexical-binding: t= is *mandatory*. The provided feature **must match the file name** (e.g., =lisp/core/foo.el= → =(provide 'core/foo)=).
- Follow *documented APIs* only—avoid private or unstable internals.
- In every =leaf= block, group settings under =:straight=, =:bind=, =:hook=, and =:custom=.
- For built-ins, always specify =:straight nil=.

*** Namespaces and naming

- Module prefixes: =core-…=, =ui-…=, =completion-…=, =orgx-…=, =dev-…=, =vcs-…=, =utils-…=
- Personal namespace: =my/...=
- *Internal-only* symbols use a double hyphen =--= (e.g., =core--restore-startup-state=)
- *Predicates* end with =-p= (e.g., =core-enabled-p=)
- Hook helpers should read clearly: =PREFIX--on-…= or =PREFIX-…-hook-function=
- Keymaps end with =-map=; hook variables end with =-hook=
- Add =;;;###autoload= **only** to public interactive commands (never to internal =--= helpers).

*** Makefile

#+begin_src text :tangle Makefile :comments no
# Makefile for Emacs config build
# - One-shot `make all`: tangle -> byte-compile (sequential)
# - Resolves EMACSD / STRAIGHT_BASE_DIR from early-init.el when available
# - Does not require a build/ directory

SHELL := /bin/sh

EMACS  ?= emacs
ORG    ?= README.org
EARLY  ?= early-init.el
INIT   ?= init.el

# Project directories (relative to repo root)
LISPDIR     ?= lisp
PERSONALDIR ?= personal

# Marker file indicating tangle completion
TANGLE_STAMP := .tangle-stamp

# ---------------------------------------------------------------------
# Resolve EMACSD by loading EARLY; fallback to user-emacs-directory
EMACSD := $(shell \
  if [ -f "$(EARLY)" ]; then \
    "$(EMACS)" --batch -Q -l "$(EARLY)" \
      --eval "(princ (expand-file-name (cond ((fboundp 'my:d) (my:d)) (t user-emacs-directory))))"; \
  else \
    "$(EMACS)" --batch -Q --eval "(princ (expand-file-name user-emacs-directory))"; \
  fi)

# Resolve STRAIGHT_BASE_DIR with several fallbacks
STRAIGHT_BASE_DIR := $(shell \
  if [ -f "$(EARLY)" ]; then \
    "$(EMACS)" --batch -Q -l "$(EARLY)" \
      --eval "(princ (expand-file-name \"straight\" (cond ((boundp 'STRAIGHT_BASE_DIR) STRAIGHT_BASE_DIR) \
                                             ((fboundp 'my:straight-base-dir) (my:straight-base-dir)) \
                                             ((boundp 'straight-base-dir) straight-base-dir) \
                                             (t user-emacs-directory))))"; \
  else \
    printf "%s" "$(EMACSD)/straight"; \
  fi)

LEAF_DIR   := $(STRAIGHT_BASE_DIR)/repos/leaf
LEAFKW_DIR := $(STRAIGHT_BASE_DIR)/repos/leaf-keywords

# (Reference only; not used as dependencies to avoid stale lists)
ELFILES  = $(shell find $(LISPDIR) $(PERSONALDIR) -type f -name '*.el' 2>/dev/null)
ELCFILES = $(patsubst %.el,%.elc,$(ELFILES))

# Minimal Org loading under -Q
EVAL_REQ_ORG := --eval "(require 'org)" \
                --eval "(require 'ob-core)" \
                --eval "(org-babel-do-load-languages 'org-babel-load-languages '((emacs-lisp . t)))"

# Optional: inject leaf paths under -Q (used by `compile-q`)
EVAL_LEAF := --eval "(add-to-list 'load-path (expand-file-name \"$(LEAF_DIR)\"))" \
             --eval "(add-to-list 'load-path (expand-file-name \"$(LEAFKW_DIR)\"))" \
             --eval "(require 'leaf)" \
             --eval "(require 'leaf-keywords)" \
             --eval "(leaf-keywords-init)"

# ---------------------------------------------------------------------
.PHONY: all tangle compile compile-q compile-personal compile-lisp \
        clean distclean show-files echo-paths echo-myd echo-sbd check-init

# Top-level: always run tangle before compile
all: tangle compile

# 1) Tangle README.org -> *.el and drop a completion stamp
$(TANGLE_STAMP): $(ORG)
	@echo "[tangle] $(ORG)"
	@"$(EMACS)" --batch -Q \
	  $(EVAL_REQ_ORG) \
	  --eval "(org-babel-tangle-file \"$(ORG)\")"
	@touch "$(TANGLE_STAMP)"

tangle: $(TANGLE_STAMP)

# 2) Compile using full init; enumerate files after tangle to avoid stale lists
compile: check-init $(TANGLE_STAMP)
	@echo "Byte-compiling .el under '$(LISPDIR)' and '$(PERSONALDIR)'..."
	@FILES=$$( { [ -d "$(LISPDIR)" ] && find "$(LISPDIR)" -type f -name '*.el' | sort; } 2>/dev/null; \
	            { [ -d "$(PERSONALDIR)" ] && find "$(PERSONALDIR)" -type f -name '*.el' | sort; } 2>/dev/null ); \
	if [ -z "$$FILES" ]; then \
	  echo "[compile] no .el files found; did tangle write to '$(LISPDIR)' or '$(PERSONALDIR)'?"; \
	  exit 2; \
	else \
	  for f in $$FILES; do \
	    echo "[byte-compile:init] $$f"; \
	    "$(EMACS)" --batch -l "$(EARLY)" -l "$(INIT)" \
	      --eval "(setq byte-compile-verbose t debug-on-error t)" \
	      --eval "(byte-compile-file \"$$f\")"; \
	  done; \
	  echo "[compile] done"; \
	fi

# Optional: compile under -Q by injecting leaf (does not load init.el)
compile-q: $(TANGLE_STAMP)
	@echo "[compile-q] -Q with STRAIGHT_BASE_DIR=$(STRAIGHT_BASE_DIR)"
	@FILES=$$( { [ -d "$(LISPDIR)" ] && find "$(LISPDIR)" -type f -name '*.el' | sort; } 2>/dev/null; \
	            { [ -d "$(PERSONALDIR)" ] && find "$(PERSONALDIR)" -type f -name '*.el' | sort; } 2>/dev/null ); \
	if [ -z "$$FILES" ]; then \
	  echo "[compile-q] no .el files found; did tangle write to '$(LISPDIR)' or '$(PERSONALDIR)'?"; \
	  exit 2; \
	else \
	  for f in $$FILES; do \
	    echo "  [byte-compile:-Q] $$f"; \
	    "$(EMACS)" --batch -Q \
	      $(EVAL_LEAF) \
	      --eval "(setq byte-compile-verbose t debug-on-error t)" \
	      --eval "(byte-compile-file \"$$f\")"; \
	  done; \
	  echo "[compile-q] done"; \
	fi

# Optional: directory-scoped compile with init.el
compile-personal: check-init $(TANGLE_STAMP)
	@echo "[compile:init] personal/"
	@find "$(PERSONALDIR)" -type f -name '*.el' -print0 2>/dev/null | xargs -0 -r -I{} \
	  "$(EMACS)" --batch -l "$(EARLY)" -l "$(INIT)" \
	    --eval "(setq byte-compile-verbose t debug-on-error t)" \
	    --eval "(byte-compile-file \"{}\")"

compile-lisp: check-init $(TANGLE_STAMP)
	@echo "[compile:init] lisp/"
	@find "$(LISPDIR)" -type f -name '*.el' -print0 2>/dev/null | xargs -0 -r -I{} \
	  "$(EMACS)" --batch -l "$(EARLY)" -l "$(INIT)" \
	    --eval "(setq byte-compile-verbose t debug-on-error t)" \
	    --eval "(byte-compile-file \"{}\")"

# Sanity check before init-based compile
check-init:
	@test -f "$(INIT)" || (echo "[error] init.el not found at: $(INIT)"; echo "        -> set INIT=/path/to/init.el or use 'make compile-q'"; exit 1)

# Helpers
show-files:
	@echo "[list] $(LISPDIR)";    { [ -d "$(LISPDIR)" ] && find "$(LISPDIR)" -type f -name '*.el' | sort; } || true
	@echo "[list] $(PERSONALDIR)"; { [ -d "$(PERSONALDIR)" ] && find "$(PERSONALDIR)" -type f -name '*.el' | sort; } || true

echo-paths:
	@echo "EMACSD=$(EMACSD)"; \
	echo "STRAIGHT_BASE_DIR=$(STRAIGHT_BASE_DIR)"; \
	echo "LEAF_DIR=$(LEAF_DIR)"; \
	echo "LEAFKW_DIR=$(LEAFKW_DIR)"; \
	echo "EARLY=$(EARLY)"; \
	echo "INIT=$(INIT)"

echo-myd:
	@echo $(EMACSD)

echo-sbd:
	@echo $(STRAIGHT_BASE_DIR)

# Cleanup
clean:
	@echo "[clean] remove *.elc under $(LISPDIR) and $(PERSONALDIR)"
	@{ [ -d "$(LISPDIR)" ] && find "$(LISPDIR)" -type f -name '*.elc' -delete; } 2>/dev/null || true
	@{ [ -d "$(PERSONALDIR)" ] && find "$(PERSONALDIR)" -type f -name '*.elc' -delete; } 2>/dev/null || true

distclean: clean
	@echo "[distclean] remove $(TANGLE_STAMP) and stray *.eln"
	@rm -f "$(TANGLE_STAMP)"
	@find . -type f -name '*.eln' -delete
#+end_src

** Core Bootstrap — Early Init & Main Init (fast, tidy, reproducible)
:PROPERTIES:
:CUSTOM_ID: core-bootstrap
:END:

*** Purpose
Stand up Emacs quickly and cleanly with a two-stage bootstrap:
- **early-init.el**: runs *before* package/UI init to speed startup, set paths, and keep artifacts under =.cache/=, =.etc/=, =.var/=.
- **init.el**: bootstraps **straight.el** + **leaf**, imports shell env (macOS-safe), dials in performance (GCMH, I/O), applies sane built-ins, then loads your modular config.

*** What this configuration does
- **Disables package.el** (we use straight.el) and widens GC/file handlers during startup, **restoring** them after init.
- Creates and wires **state directories**: =.cache/=, =.etc/=, =.var/=, =eln-cache/=, Treesit build/grammars, URL/EWW state.
- On **macOS**, prefers Homebrew toolchain: updates =PATH=, =LIBRARY_PATH=, and selects the highest =gcc-N= for native compilation.
- Puts native-compile ELN outputs under =.cache/eln-cache= and silences async warnings.
- Preconfigures **no-littering** and **straight.el** bases.
- Turns **backups** and classic **auto-save** *off* early (later uses auto-save-visited).
- Applies early **UI defaults** (hide bars, pixel-precision scrolling, frame alists) to avoid flicker.
- In =init.el=: sets URL state paths *before* =url.el= loads, bootstraps **straight.el**, imports shell env (exec-path-from-shell), enables **GCMH**, raises =read-process-output-max= during start, enables useful built-ins (line numbers, electric-pair, window-divider), and loads **modules.el**.
- Prints a friendly **startup time + GC count** message after init.

*** Module map (where things live)
| Module file  | Role |
|--------------+------|
| =early-init.el= | Pre-package/bootstrap phase (GC, paths, UI, native-comp, macOS toolchain) |
| =init.el=       | straight.el bootstrap, env import, performance knobs, core built-ins, modules entry |

*** How it works (flow)
1. **early-init.el** runs first:
   - Disable =package.el=, set base dirs, create cache/var/etc trees.
   - On macOS, augment env for gcc/libgccjit; set =CC= to the newest gcc.
   - Route native-comp ELN to =.cache/eln-cache=; widen GC; nil out file handlers; hide GUI chrome; set frame defaults.
2. **init.el**:
   - Establish URL state dirs, then require =url= to support straight bootstrap.
   - Bootstrap **straight.el**; install **leaf** et al.; initialize leaf keywords.
   - Import shell env (macOS GUI/daemon safe) and complement =exec-path= with Homebrew bins if needed.
   - Apply performance tweaks (GCMH; temporarily enlarge =read-process-output-max=).
   - Enable built-ins (relative line numbers, electric-pair, window divider, pixel scroll), unbind risky keys, enable auto-save-visited.
   - Load personal overrides (optional) and then **modules** from =lisp/=.
   - Announce startup timing.

*** Key settings (reference)
- =package-enable-at-startup= :: =nil= — avoid double package init.
- =native-comp-eln-load-path= :: Points to =.cache/eln-cache=.
- =no-littering-etc/var-directory= :: Pointed to =.etc/= and =.var/= *early*.
- =straight-base-dir= :: =.cache/= (keeps repos/build under cache).
- =gc-cons-threshold / gc-cons-percentage= :: Maximized during startup; restored to ~16MB / 0.1.
- =file-name-handler-alist= :: Temporarily =nil= for speed; restored on startup hook.
- =make-backup-files / auto-save-default= :: Disabled early; =auto-save-visited-mode= enabled later with short interval.
- macOS toolchain :: =PATH=, =LIBRARY_PATH=, and =CC= adjusted to prefer Homebrew gcc/libgccjit.

*** Usage tips
- **Keep it tidy**: put local state (cookies, URL cache, EWW data) under =.var/url= and =.var/eww= to make your config directory portable.
- **Speed profiling**: combine this bootstrap with module-level timing in =modules.el= to pinpoint slow loads.
- **Daemon & GUI**: exec-path import is guarded for macOS so both daemon and GUI sessions inherit your shell env reliably.

*** Troubleshooting
- *“straight bootstrap failed”* → Check network, and that =url.el= was required first (it is). Review *Messages* for the installer’s error.
- *Native-comp not used* → Ensure Emacs was built with native compilation and that =native-comp-eln-load-path= points at a writable =.cache/eln-cache=.
- *Weird toolchain on macOS* → Confirm Homebrew prefix detection (=/opt/homebrew= or =/usr/local=) and that =libgccjit= is installed.
- *Backups disappeared* → That’s by design (backups off). If you want them, flip settings later or use the backup helper in =utils/utils-backup.el=.

  ;;
  ;; Copyright (c) 2021-2025
  ;; Author: YAMASHITA, Takao
  ;; License: GNU GPL v3 or later
  ;;
  ;;; Commentary:
  ;; Core bootstrap executed before regular init.
  ;; Category: core
  ;;
  ;; - Disable package.el (we use straight.el + leaf)
  ;; - Fast startup: widen GC/file-name-handlers and restore after startup
  ;; - Keep artifacts under .cache/ .etc/ .var/ (native-compile, no-littering, straight)
  ;; - macOS: prefer Homebrew gcc/libgccjit (PATH/LIBRARY_PATH/CC)
  ;; - Backups OFF; classic auto-save OFF (use auto-save-visited in init)
  ;; - Early UI: frame defaults & hide GUI bars to avoid flicker

  (eval-when-compile (require 'subr-x))

  ;;;; Internal helpers (core namespace) -----------------------------------------
  (defun core--ensure-directory-exists (dir)
    "Ensure DIR exists, creating it if necessary."
    (unless (file-directory-p dir)
      (condition-case err
          (make-directory dir t)
        (error (warn "Failed to create directory: %s (%s)"

*** Core Layer — General / Editing / History / Switches / Tools / Custom
:PROPERTIES:

**** Purpose
Provide a coherent **core layer** for everyday editing and operations:
- **Meow** modal editing with a structured **Space-leader** hierarchy and **Which-Key** discoverability.
- Practical **editing UX** (Paredit/Puni/Show-Paren/Undo-Fu/Vundo, Dired subtree/filter, auto-revert).

- Developer/authoring helpers (Treesit install pathing, Org fold helpers, window split toggle, VSCode jump).
- Stable **Customize** storage under `.etc/custom.el` with snapshot/open helpers.

**** What this configuration does
- Sets up **Meow** normal/motion states, ESC to leave insert, and **leader maps** (`SPC b/w/p/...`) implemented as real prefix keymaps for clean which-key labels.
- Installs **Hydra** for text scaling (`core-hydra-text-scale`) and wires common global bindings (macOS-like `s-` chords, Consult shortcuts).
- Routes TRAMP, Transient, Saveplace/Recentf/Savehist storage into `.var/…`; enables **auto-save-visited** (real files, no `#…#`).
- Provides **Switches** (`my:use-ui`, `my:use-lsp`) with autoloaded enable functions and graceful fallback logging.
- Adds **Treesit** grammar install advice to honor `my:d:treesit` paths and ensures directories exist.
- Centralizes **Customize** output at `.etc/custom.el`; offers `my/custom-open` and `my/custom-dump-current`.

   - History/persistence files and TRAMP/Transient are redirected into `.var/…`.
   - If `my:use-ui` ≠ `none`, try enabling `doom`/`nano` bundle; logs if present but failed/unavailable.
   - Enable chosen **LSP backend** (`eglot`/`lsp`) with presence checks.
2. **Editing session**
   - **Meow** is globally enabled; **leader groups** under `SPC` expose buffer/window/project/git/code/error/toggle/org/misc/ai/session/help.
   - **Which-Key** displays leader group labels like “SPC b → buffers”.
   - **Hydra** provides quick text scaling; global bindings keep macOS-like convenience and Consult jumps.
3. **Day-to-day**
   - **Auto-save-visited** writes buffers regularly; **auto-revert** picks up external changes quietly.
   - Use **Treesit** helpers to install grammars into `my:d:treesit` with ensured directories.
   - For Customize-driven tweaks, use `my/custom-open` and **snapshot** current settings/faces with `my/custom-dump-current`.

**** Key settings (reference)
- =auto-save-visited-interval= :: =1= second; =auto-save-default= :: =nil= (no `#…#`).
- =tramp-persistency-file-name= :: → `.var/tramp` ; =tramp-auto-save-directory= :: → `.var/tramp-autosave`.
- =save-place-file= :: → `.var/saveplace` ; =recentf-save-file= :: → `.var/recentf` ; =savehist-file= :: → `.var/history`.
- =transient-*-file= :: → `.var/transient/{history,levels,values}.el`.
- Meow :: `meow-global-mode t`, ESC leaves insert, motion-state overwrites for read-only buffers.
- Which-Key :: `which-key-idle-delay = 0.4`, labels added for `SPC` group heads.

**** Usage tips
- **Leader discovery**: Type =SPC= and pause to see groups (buffers/windows/project/git/code/errors/toggles/org/misc/ai/session/help).
- **Project grep**: =SPC p s= (or =SPC s g=) calls `consult-ripgrep` at project root; falls back to prompting if no project.
- **Text scale**: =SPC z= (or =C-c z=) opens the scaling Hydra.
- **Customize snapshot**: Run =M-x my/custom-dump-current= to persist curated variables and faces into `.etc/custom.el`.

**** Troubleshooting
- *Customize file not found / not loaded* → `my/custom--ensure-file` creates and loads `.etc/custom.el`; ensure `.etc/` exists and is writable.
- *Switch failed to enable UI/LSP* → See `[switches] …` messages; verify module files (`ui/ui-*.el`, `dev/dev-*.el`) are on `load-path`.
- *GPG warning at startup* → Set `GPG_KEY_ID` (used by `auth-source`/plstore). `PASSWORD_STORE_DIR` defaults to `.var/password-store/`.
- *Meow expands in the wrong major mode* → Adjust `meow-expand-exclude-mode-list`; motion-state overwrites are provided for read-only buffers.
- *Treesit grammar not installed in the right place* → `my/treesit--call-with-outdir` forces `my:d:treesit` as OUT-DIR; ensure `my:d:treesit`/`my:d:treesit-build` are defined.

  (defun my/toggle-window-split ()
    "Toggle between horizontal and vertical split for two windows."
    (interactive)
    (when (= (count-windows) 2)
      (let* ((this-buf (window-buffer))
             (next-buf (window-buffer (next-window)))
             (this-edges (window-edges))
             (next-edges (window-edges (next-window)))
             (split-vert (= (car this-edges) (car next-edges)))
             (split-fn (if split-vert #'split-window-horizontally #'split-window-vertically)))
        (delete-other-windows)
        (funcall split-fn)
        (set-window-buffer (selected-window) this-buf)
        (set-window-buffer (next-window) next-buf)
        (select-window (selected-window)))))

  (defun my/find-keybinding-conflicts ()
    "Find and display conflicting keybindings in active keymaps."
    (interactive)
    (let ((conflicts (make-hash-table :test 'equal))
          (maps (current-active-maps t))
          (buffer-name "*Keybinding Conflicts*"))
      (dolist (map maps)
        (map-keymap
         (lambda (key cmd)
           (when (commandp cmd)
             (let ((desc (key-description (vector key)))
                   (existing (gethash desc conflicts)))
               (puthash desc (delete-dups (cons cmd existing)) conflicts))))
         map))
      (with-current-buffer (get-buffer-create buffer-name)
        (read-only-mode -1)
        (erase-buffer)
        (insert "* Keybinding Conflicts *\n\n")
        (maphash (lambda (key cmds)
                   (when (> (length cmds) 1)
                     (insert (format "%s => %s\n" key (mapconcat #'symbol-name cmds ", ")))))
                 conflicts)
        (read-only-mode 1))
      (pop-to-buffer buffer-name)))

    "Advice ORIG-FN to force OUT-DIR to `my:d:treesit` when omitted."

      (setf (nth 6 args*) out-dir)

*** UI Layer — Fonts / Themes / Modelines / Icons / Windows / Utils
:PROPERTIES:
:CUSTOM_ID: ui-layer
:END:

**** Purpose
Deliver a cohesive **UI layer** with:
- **Fonts**: sane defaults per-OS, emoji font, optional ligatures (guarded).
- **Themes**: =ef-themes= + optional =spacious-padding=, with **face normalization** for Emacs 30+.
- **Modelines**:
  - **Doom modeline** bundle (=ui-doom-modeline.el=) w/ nerd-icons.
  - **Nano-like modeline** (=ui-nano-modeline.el=): compact header-/mode-line with LSP, clock, battery.
- **Icons**: nerd-icons for minibuffer/dired, safe fallbacks when Nerd Font unavailable.
- **Windows**: zoom-centered workflow, desktop save/restore, winner-mode, quick save/restore layout.
- **Utilities**: Treemacs defaults, minions, time/battery display, pbcopy on macOS.

**** What this configuration does
- Applies **font** setup at startup/daemon frame creation; reports applied fonts (default/variable/emoji).
- Loads an **ef-theme** (GUI: =ef-frost=; TTY: =deeper-blue=) unless one is already active; enables **spacious-padding** on GUI.
- Enables either **doom-modeline** (`my/ui-enable-doom`) or **nano-like modeline** (`my/ui-enable-nano`) depending on switches.
- Activates **nerd-icons** integrations **only** when a Nerd Font is detected; otherwise degrades gracefully.
- Sets up **zoom-mode**, **desktop.el** (saved in `.var/desktop/`), and **winner-mode** bindings (=M-[= / =M-]=).
- Turns on **minions**, **display-time**, and **battery** (when available); wires Treemacs for GUI.

**** Module map (where things live)
| Module file              | Role                                                                                       |
|--------------------------+--------------------------------------------------------------------------------------------|
| =ui/ui-font.el=          | Default/variable/emoji fonts, guarded ligatures, frame-hooks                               |
| =ui/ui-theme.el=         | ef-themes load, spacious-padding, face normalization on theme enable/startup               |
| =ui/ui-doom-modeline.el= | Doom modeline bundle (nerd-icons); =my/ui-enable-doom= to activate                         |
| =ui/ui-nano-modeline.el= | Nano-like header-/mode-line with LSP/Flymake/clock/battery, global minor mode + timer      |
| =ui/ui-icons.el=         | nerd-icons (completion/dired), diredfl, helper segments for nano-modeline                  |
| =ui/ui-utils.el=         | Minions, Treemacs defaults, time/battery display, pbcopy (macOS)                           |
| =ui/ui-window.el=        | zoom-mode, desktop save/restore, winner-mode, ad-hoc save/restore window layout           |

**** How it works (flow)
1. **Startup**
   - Fonts are applied (`my/font-setup`) after init or on new daemon frames.
   - A theme is loaded if none was chosen by user; **spacious-padding** is enabled on GUI.
   - Face attributes normalized post-theme via =enable-theme-functions= hook.
2. **Modeline choice**
   - When **Doom** is chosen: `doom-modeline-mode 1` with compact settings (height 28, project-truncation, no minor modes).
   - When **Nano-like** is chosen: enable =my/ui-nano-modeline-mode= globally.
     - Composes Left/Center/Right segments, supports **Meow** state, VC branch, LSP/Eglot ✓, Flymake summary, pos/percent, clock, battery.
     - Places UI in **header-line** by default (toggle with `my/ui-nano-modeline-toggle-placement`).
     - Starts a **quiet 60s refresh timer** (no echo/message-log).
3. **Icons**
   - If a preferred **Nerd Font** is present and icons are enabled: turn on nerd-icons for completion/dired; otherwise skip silently.
4. **Windows & Session**
   - **zoom-mode** centers focus; **desktop.el** persists sessions under `.var/desktop/`; **winner-mode** binds undo/redo.
   - Helpers: `my/save-window-layout` / `my/restore-window-layout` snapshot ad-hoc layouts.

**** Key settings (reference)
- Fonts :: =my:font-default= / =my:font-size= (pt = size/1.0), =my:font-alt= (variable-pitch), =my:emoji-font=.
- Themes :: Default GUI theme =ef-frost=; TTY =deeper-blue=; normalized faces: =ui:normalize-face-attrs=.
- Doom modeline :: =doom-modeline-height=28=, file-name style =truncate-with-project=, word-count on, minor-modes off.
- Nano-like :: Header-line placement (=my/ui-nano-use-header-line t=), center width cap (=my/ui-nano-center-max=, default 44),
               show VC/LSP/Flymake/Clock/Battery toggles (=my/ui-nano-show-*= vars).
- Icons :: Detection via preferred list (=my:icons-preferred-fonts=); guarded enablement.
- Windows :: =zoom-size='(0.62 . 0.62)=, =desktop-dirname= → `.var/desktop/`, winner-mode global.

**** Usage tips
- **Switch modeline** quickly:
  - Doom: =M-x my/ui-enable-doom=
  - Nano-like: =M-x my/ui-enable-nano= (or =my/ui-nano-modeline-mode=)
  - Toggle header ↔ mode: =M-x my/ui-nano-modeline-toggle-placement=
- **Fonts**:
  - Show current: =M-x my/show-current-font=
  - Override defaults via =my:font-default= / =my:font-size= / =my:font-alt= / =my:emoji-font=
- **Icons**:
  - Install a Nerd Font family and ensure it’s active for your frame; icons auto-enable if detected.
- **Windows**:
  - Undo/redo window layouts: =M-[= / =M-]=
  - Save/restore custom layout: =M-x my/save-window-layout= / =M-x my/restore-window-layout=

**** Troubleshooting
- *Icons not showing* → Confirm a **Nerd Font** is installed & used by the frame; check =my:icons-enabled= and preferred families.
- *Nano-like modeline doubles with upstream nano-modeline* → The mode removes upstream hooks; ensure no external re-enable.
- *LSP indicator empty* → Confirm Eglot/lsp-mode loaded and active for the buffer; indicators appear only when connected.
- *Clock/Battery missing* → Clock respects =my/ui-nano-show-clock=; battery requires `(display-battery-mode 1)` **and** OS support.
- *Theme face missing color* → The **face normalizer** sets nil → `unspecified`; extend =ui:normalize-face-attrs= as needed.

    "Pad string S with spaces on LEFT and RIGHT."

            (make-string (or right 0) ?\s)))

                       ((fboundp 'meow--state) (meow--state))

    "Return current VC branch name (or empty)."
    (when (and my/ui-nano-show-vc (boundp 'vc-mode) vc-mode)

             (branch (string-trim (replace-regexp-in-string "^ *[^:]+:" "" s))))
        (when (> (length branch) 0)
          (concat
           (my/ui-nano--icon "call_split" "⎇" 'my/ui-nano-accent) " "

    "Return pretty buffer name (with modified marker)."
    (let* ((name (buffer-name)))
      (if (buffer-modified-p)
          (concat (my/ui-nano--pro "●" 'my/ui-nano-warning 0 1)

    "Return line:col and percent of buffer."
    (let* ((lc (format "%d:%d" (line-number-at-pos) (current-column)))
           (ratio (if (> (buffer-size) 0)
                      (/ (float (point)) (buffer-size))
                    0.0))
           (pc (format "%d%%%%" (round (* 100 ratio)))))
      (concat (my/ui-nano--pro lc 'my/ui-nano-face 0 1)
              (my/ui-nano--pro pc 'my/ui-nano-accent 0 1))))

  (defun my/ui-nano--flymake-summary ()
    "Return Flymake error/warn/notes summary if enabled."
    (when (and my/ui-nano-show-flymake (bound-and-true-p flymake-mode))
      (let ((diags (flymake-diagnostics (point-min) (point-max)))
            (e 0) (w 0) (n 0))
        (dolist (d diags)
          (pcase (flymake-diagnostic-type d)
            (':error   (setq e (1+ e)))
            ('error    (setq e (1+ e)))
            (':warning (setq w (1+ w)))
            ('warning  (setq w (1+ w)))
            (_         (setq n (1+ n)))))
        (when (> (+ e w n) 0)
          (concat
           (when (> e 0)
             (concat (my/ui-nano--icon "error" "E" 'my/ui-nano-warning)
                     (my/ui-nano--pro (format ":%d " e) 'my/ui-nano-warning)))
           (when (> w 0)
             (concat (my/ui-nano--icon "warning" "W" 'my/ui-nano-accent)
                     (my/ui-nano--pro (format ":%d " w) 'my/ui-nano-accent)))
           (when (> n 0)
             (my/ui-nano--pro (format "N:%d " n) 'my/ui-nano-face)))))))

  (defun my/ui-nano--lsp-eglot-status ()
    "Return a compact LSP/Eglot status string (✓ when connected)."
    (when my/ui-nano-show-lsp
      (cond
       ;; Eglot
       ((featurep 'eglot)
        (let ((srv (and (fboundp 'eglot-current-server)
                        (eglot-current-server))))
          (if srv
              (concat
               (my/ui-nano--icon "language" "LSP" 'my/ui-nano-ok)
               (my/ui-nano--pro "✓" 'my/ui-nano-ok 0 1))
            "")))
       ;; lsp-mode
       ((featurep 'lsp-mode)
        (let* ((active (and (bound-and-true-p lsp-mode)
                            (ignore-errors (length (bound-and-true-p lsp--buffer-workspaces))))))
          (if (and active (> active 0))
              (concat
               (my/ui-nano--icon "language" "LSP" 'my/ui-nano-ok)
               (my/ui-nano--pro "✓" 'my/ui-nano-ok 0 1))
            "")))
       (t ""))))

          (concat sym (my/ui-nano--pro (format "%s" pct) 'my/ui-nano-face 0 1))))))

  (defun my/ui-nano--format ()
    "Return a format list for `mode-line-format`/`header-line-format`."
    (list
     ;; Left
     '(:eval
       (let* ((left (or (my/ui-nano--seg-left) ""))
              (hpad (cdr my/ui-nano-padding)))
         (propertize (my/ui-nano--pad left 0 (floor (* 2 hpad)))
                     'face 'my/ui-nano-face)))
     ;; Center (true centering with align-to + measured width)
     '(:eval
       (let* ((center (or (my/ui-nano--seg-center) ""))
              (w (max 0 (string-width center)))
              (hpad (cdr my/ui-nano-padding)))

          (propertize
           " " 'display `((space :align-to (+ center (- (/ ,w 2))))))
          (propertize (my/ui-nano--pad center 0 (floor (* 2 hpad)))
                      'face 'my/ui-nano-face))))
     "%="                                          ; filler to right
     ;; Right
     '(:eval
       (let* ((right (or (my/ui-nano--seg-right) "")))
         (propertize right 'face 'my/ui-nano-face)))))

    "Quietly refresh the modeline without echo-area or log output."

          (message-log-max nil))   ;; suppress *Messages*
      (force-mode-line-update t)))

    (setq my/ui-nano--tick-timer

    "Stop the periodic refresh timer if running."
    (when (timerp my/ui-nano--tick-timer)
      (cancel-timer my/ui-nano--tick-timer))
    (setq my/ui-nano--tick-timer nil))

  (defun my/ui-nano--install ()
    "Install nano-like line according to `my/ui-nano-use-header-line`."
    (setq my/ui-nano--saved-mode-line   mode-line-format
          my/ui-nano--saved-header-line header-line-format)

        (progn
          (setq header-line-format (my/ui-nano--format))
          (setq mode-line-format   nil))
      (setq mode-line-format   (my/ui-nano--format))
      (setq header-line-format nil)))

*** Completion Stack — Orderless + Corfu + Vertico + Consult + Embark
:PROPERTIES:
:CUSTOM_ID: completion-stack
:END:

**** Purpose
Provide a modern, low-friction completion UX that combines:
- **Orderless** matching everywhere (keep **partial-completion** for files/paths).
- **Corfu** for in-buffer popup completion (CAPF-driven), with **CAPE** to compose a robust source pipeline.
- **Vertico** for minibuffer candidates, plus **Marginalia** annotations and optional **posframe** UI.
- **Consult** for xref/grep/jump integrations and a clean fallback **completion-in-region** UI.
- **Embark** actions with live preview in collect buffers.
- Built-in **completion-preview-mode** (“ghost text”) where a popup UI is not active.

**** What this configuration does
- Makes **TAB** smart (`tab-always-indent='complete`): indent first; if already aligned, trigger completion.
- Enables **completion-preview-mode** in typical editing modes so ghost-text appears when Corfu isn’t used.
- Picks **completion UI dynamically**:
- If `corfu-mode` is active in the buffer → use `corfu-completion-in-region` (popup).
- Otherwise → use `consult-completion-in-region` (minibuffer), which cooperates with the ghost-text preview.
- Unifies CAPF via **CAPE**:
- Priority: **eglot/LSP** (if present) → **file** → **keyword** → **dabbrev**.
- The composed CAPF is installed buffer-locally by `my/setup-capf` for `prog-/text-/conf-mode`.
- Keeps **file/category** behavior sensible:
- `completion-styles` = `(orderless basic)`
- `completion-category-overrides.file` = `(partial-completion)` for natural path segment completion.

**** Module map (where things live)
| Module file                                | Role                                                                 |
|--------------------------------------------+----------------------------------------------------------------------|
| =completion/completion-core.el=            | Global defaults (Orderless, TAB policy, completion-preview-mode)     |
| =completion/completion-corfu.el=           | Corfu popup, CAPE pipeline, dynamic completion-in-region switch      |
| =completion/completion-vertico.el=         | Vertico minibuffer UI, Marginalia, optional vertico-posframe         |
| =completion/completion-consult.el=         | Consult glue (xref, grep, fallback in-region UI)                     |
| =completion/completion-embark.el=          | Embark actions + consult preview in collect buffers                  |
| =completion/completion-icons.el=           | Nerd icons (ibuffer/Marginalia) and Corfu kind icons (via kind-icon) |

**** How it works (flow)
1. You start typing:
- **Minibuffer** → Vertico + Orderless filter candidates; Marginalia annotates.
- **Editing buffer** → If =corfu-mode= is active, CAPF candidates appear in a Corfu popup.
2. Press =TAB=:
- If indentation is needed, it indents; otherwise it completes.
3. If Corfu is **inactive** in the current buffer:
- **completion-preview-mode** shows ghost-text,
- and =completion-in-region-function= uses **Consult** to perform the completion in the minibuffer.
4. Source selection:
- If Eglot is managing the buffer, LSP CAPF is first; otherwise CAPE falls back to file/keyword/dabbrev.

**** Key settings (reference)
- =tab-always-indent= :: ='complete — indent then complete.
- =completion-cycle-threshold= :: Show cyclic TAB for small sets (e.g., =3=).
- =completion-styles= :: =(orderless basic)=.
- =completion-category-overrides= ::
- =file= → =(partial-completion)=
- =symbol=, =command= → =orderless=
- =completion-preview-mode= :: Enabled for =text-/prog-/conf-mode=.
- =completion-in-region-function= :: Switches between Corfu/Consult by checking =corfu-mode=.

**** Usage tips
- **Fuzzy-ish multi-term matching**: Type space-separated patterns with Orderless (e.g., =rg path=).
- **Path completion**: For nested directories, partial-completion is friendlier than pure fuzzy.
- **Embark actions**: =C-.= (or your binding) to act on the current candidate; in collect buffers, live preview is enabled.
- **Icons**: Nerd/Kind icons enrich minibuffer and Corfu popups when available.

**** Troubleshooting
- *“Ghost text overlaps the popup”* → This setup disables Corfu’s own preview (=corfu-preview-current=nil=) to avoid clashes.
- *“No LSP suggestions”* → Ensure Eglot is active; =my/setup-capf= will automatically prefer LSP CAPF when available.
- *“Too many dabbrev hits”* → Reorder or remove =cape-dabbrev= inside =cape-super-capf= in =completion-corfu.el=.
- *“Minibuffer too crowded”* → Tweak =vertico-count= or disable =vertico-posframe= if desired.

*** Org Layer — Core / Export / Extensions / Visual

**** Purpose
A cohesive **Org layer** that covers daily notes, task/agenda management, Zettelkasten, and publishing.
- **Core**: directory scaffolding, Agenda auto-discovery, TODO flow, Capture, Refile, central Archive.
- **Export**: Hugo / Markdown / LaTeX via ox backends.

**** What this configuration does

- **TODO flow**: `TODO → SOMEDAY → WAITING | DONE / CANCELED@` (the `@` keeps a note on cancel).
- **Capture** templates: GTD Inbox, Notes, Journal, Meeting.
- **Refile** targets: current file (maxlevel 3), open Org buffers (maxlevel 1), agenda files (maxlevel 3).
- **Archive**: central archive at `archive.org::`.
- **Export**: ox-hugo (front matter in TOML), Markdown editing/preview, indirect region editing.
- **Extensions**: org-journal dir, org-roam directory & DB + autosync, org-download image dir, toc-org on Org/Markdown, org-cliplink binding.
- **Visual**: org-modern tweaks for indentation, markers, icons, agenda separators and time grid.

*** VCS Layer — Magit / Forge / diff-hl (Fringe)

**** Purpose
A lean, reliable **version control layer** for day-to-day Git work:
- **Magit** for full-featured Git porcelain.
- **Forge** to work with GitHub/GitLab issues/PRs directly from Magit.
- **diff-hl** as the sole, lightweight fringe indicator for line changes.

**** What this configuration does
- Binds **Magit Status** to =C-x g=.
- Enables **diff-hl** in programming buffers, text buffers, and Dired; auto-refreshes after Magit actions.
- Configures **Forge** to store its SQLite DB under the no-littering var directory:
  =…/var/forge/forge-database.sqlite= (ensures the =forge/= subdirectory exists).

**** Module map (where things live)
| Module file          | Role                                                                                   |
|----------------------+----------------------------------------------------------------------------------------|
| =vcs/vcs-magit.el=   | Magit entry; keybinding (=C-x g → magit-status=)                                       |
| =vcs/vcs-gutter.el=  | diff-hl in fringe for prog/text/Dired; integrates with Magit’s refresh cycle           |
| =vcs/vcs-forge.el=   | Forge init after Magit; database path under no-littering and directory creation        |

**** How it works (flow)
1. **Status & commits**
   Hit =C-x g= → Magit’s status buffer provides staging, commits, rebases, logs, etc.
2. **Fringe change indicators**
   Open any prog/text/Dired buffer → =diff-hl-mode= (or =diff-hl-dired-mode=) highlights added/modified lines.
   After Magit operations, =diff-hl-magit-post-refresh= keeps indicators in sync.
3. **Forge integration**
   Opening a Magit repo with a GitHub/GitLab remote → Forge uses its SQLite DB at
   =no-littering-var-directory/forge/forge-database.sqlite= to cache issues/PR metadata.

**** Key settings (reference)
- Magit :: Keybinding → =("C-x g" . magit-status)=
- diff-hl :: Hooks → =(prog-mode text-mode dired-mode)=; Magit integration → =(magit-post-refresh . diff-hl-magit-post-refresh)=; borders off → =(diff-hl-draw-borders . nil)=
- Forge DB :: =(expand-file-name "forge/forge-database.sqlite" no-littering-var-directory)=
  Directory ensured via =my/ensure-directory-exists=

**** Usage tips
- **Stage with precision**: =s= on a hunk in Magit; use =e= to edit the hunk before committing.
- **Navigate changes fast**: In buffers with =diff-hl=, use your usual next/prev hunk keys (e.g., via =diff-hl-diff-goto-hunk=).
- **Work on PRs in-place**: From Magit, invoke Forge commands (e.g., to list issues/PRs) and comment or checkout PR branches without leaving Emacs.

**** Troubleshooting
- *Fringe marks don’t update after a commit* → Ensure =diff-hl= received =magit-post-refresh= (this config wires it via a hook).
- *Forge shows no issues/PRs* → Check that the remote host is supported and credentials are configured; confirm the DB path is writable.
- *No fringe indicators in Dired* → Verify =diff-hl-dired-mode= is enabled (it’s added via the Dired hook here).

*** Search, Navigation & QoL Utilities — Ripgrep/Consult, Dumb-Jump, EWW helpers, Async, Backups, Meow Read-Only, Scratch
:PROPERTIES:
:CUSTOM_ID: utils-search-nav
:END:

**** Purpose
Provide fast, modern **project search & navigation** and a handful of quality-of-life helpers:
- **Ripgrep-first xref** via **dumb-jump** (as an Xref backend) for lightweight “go to definition” across projects.
- **EWW** convenience: one-shot web search with auto-highlight and an **image toggle** (reloads the page).
- **Async runner** that safely executes functions and logs errors (no UI freeze).
- **Backup janitor** that prunes stale backups under a **no-littering**-aware var directory.
- **Smart buffer closing**, **Meow read-only behavior** (Space to scroll), and **auto-recreate \*scratch\***.

**** What this configuration does
- Registers **dumb-jump** on `xref-backend-functions`, preferring **ripgrep** (`rg`) as the searcher.
- Sets **EWW** defaults:
  - Search engine (DuckDuckGo HTML), download dir, expanded history, and a dedicated EWW var dir.
  - `M-x my/eww-search` opens results and **starts isearch** with your query; `my/eww-toggle-images` flips `shr-inhibit-images` and reloads.
- Adds `my/safe-run-async` to run tasks in the future tick while catching and logging errors.
- Creates `backup/` (under `no-littering`’s var dir when available) and **deletes backups older than 7 days** on startup (configurable).
- Defines `my/kill-buffer-smart` (kill buffer & window when appropriate).
- When **Meow** is present and the buffer is read-only or derived from viewing modes, switches to **motion state** and maps:
  - `SPC`/`S-SPC` to scroll, `q` to `quit-window`, and `,`/`M-SPC` to the keypad/leader.
- Automatically **recreates \*scratch\*** after it’s killed if `my:scratch-auto-recreate` is non-nil.

**** Module map (where things live)
| Module file                         | Role                                                                 |
|-------------------------------------+----------------------------------------------------------------------|
| =utils/search-nav.el=              | Dumb-Jump xref setup, EWW search/image toggle, EWW data locations    |
| =utils/utils-async.el=             | Safe async runner (`my/safe-run-async`)                              |
| =utils/utils-backup.el=            | Backup directory + stale backup cleanup (startup hook)               |
| =utils/utils-functions.el=         | `my/kill-buffer-smart`                                               |
| =utils/utils-meow-readonly.el=     | Meow motion state for read-only/view buffers; Space scrolls          |
| =utils/utils-scratch.el=           | Auto-recreate `*scratch*`; option `my:scratch-auto-recreate`         |

**** How it works (flow)
1. **Navigation**
   - Xref queries (e.g., `M-.`) can fall back to **dumb-jump** which uses **ripgrep** across your project.
2. **Web search**
   - `M-x my/eww-search` → opens DuckDuckGo results; the term is yanked into isearch so hits are highlighted immediately.
   - `M-x my/eww-toggle-images` → toggles images and reloads the current page.
3. **Background tasks**
   - Wrap work with `my/safe-run-async` to avoid blocking and to log exceptions.
4. **Backups**
   - On startup, backups older than 7 days in the managed dir are pruned.
5. **Read-only buffers**
   - If Meow is loaded, read-only/help/special buffers flip to **motion**; `SPC` pages down, `S-SPC` up, `q` quits.
6. **Scratch lifecycle**
   - Killing `*scratch*` silently schedules a fresh one with `lisp-interaction-mode`.

**** Key settings (reference)
- **Dumb-Jump**
  - =dumb-jump-force-searcher= :: ='rg
  - =dumb-jump-prefer-searcher= :: ='rg
- **EWW**
  - =eww-search-prefix= :: DuckDuckGo HTML endpoint (safe, minimal)
  - =eww-bookmarks-file=, =eww-cache-directory= :: placed under a dedicated EWW var dir
  - =eww-history-limit= :: =200=
  - Commands: =my/eww-search=, =my/eww-toggle-images=
- **Backups**
  - Startup cleanup: =my/delete-old-backups= (default 7 days; call with prefix arg to override)
- **Meow (optional)**
  - Modes considered “viewing”: =view-mode=, =help-mode=, =special-mode= (customize via =my:meow-readonly-derive-modes=)
- **Scratch**
  - =my:scratch-auto-recreate= :: non-nil → auto-recreate; alias =*scratch-auto-recreate= kept for compatibility

**** Usage tips
- **Jumping around**: If your project lacks tags or an LSP server, dumb-jump + ripgrep is a quick, zero-config way to navigate.
- **Web reading**: Turn off images on slow links or when you only need text; toggle back when needed.
- **Batch cleanups**: Call `(my/delete-old-backups 30)` to keep only last-month backups.
- **Meow users**: In log viewers, help buffers, or read-only modes, just use `SPC`/`S-SPC` to page like a pager.

**** Troubleshooting
- **“Ripgrep not found”** → Install `rg` in PATH; dumb-jump defaults are tuned for `rg`.
- **EWW cache/bookmarks not created** → Ensure the configured EWW var dir is writable; it’s created on first use.
- **Space doesn’t scroll in read-only** → Confirm Meow is loaded; this feature activates only when `featurep 'meow`.
- **Backups not pruning** → Verify the backup dir exists and Emacs had permission to delete files; run `M-x my/delete-old-backups`.

      buf))

    (when (and my:scratch-auto-recreate
               (string= (buffer-name buf) "*scratch*"))
      (run-at-time 0 nil #'my/create-scratch-buffer)))

            (lambda ()
              (my/kill-scratch-buffer-advice (current-buffer))))

    (let* ((var-dir (cond
                     ((boundp 'no-littering-var-directory) no-littering-var-directory)
                     (t my--fallback-var-dir)))
           (dir (expand-file-name "backup/" var-dir)))
      (unless (file-directory-p dir)
        (make-directory dir t))
      dir))

  (defun my/delete-old-backups (&optional days)
    "Delete backup files older than DAYS (default: 7)."
    (interactive "P")
    (let* ((ndays (or (and (numberp days) (max 1 days)) 7))
           (threshold (- (float-time (current-time)) (* ndays 24 60 60)))
           (backup-dir (my--backup-dir)))
      (dolist (file (directory-files backup-dir t))
        (when (and (file-regular-p file)
                   (< (float-time (file-attribute-modification-time
                                   (file-attributes file)))
                      threshold))
          (ignore-errors (delete-file file))))))

** Personal Config — Identity, Fonts, UI/LSP Switches, macOS IME & Apple Music
:PROPERTIES:
:CUSTOM_ID: user-config

*** Purpose
Centralize personal identity, font choices, and device-specific tweaks in one place. On macOS, add quality-of-life integrations:
- **Identity & directories** (cloud/org/blog) with safe directory creation.
- **Fonts** (mono / variable / emoji) and **UI/LSP** switches respected by the core.
- **Org defaults** (agenda roots kept in sync with your cloud `org/`).
- **macOS IME** via **sis + macism** (optional).
- **Apple Music** control with a **Hydra** and **Meow leader** binding.

*** What this configuration does
- Sets your **name/email**, enables convenient security options (e.g., `plstore` caching).
- Picks default **fonts** & size, and sets high-level **switches** (`my:use-ui`, `my:use-lsp`) for the rest of the config.
- Defines **cloud directories** (`Documents/org`, `…/mysite`) and ensures they exist; keeps `org-directory` synced.
- Builds **agenda files** from `org-directory` while excluding `archives`.
- Prunes sensitive/system paths out of **`load-path`** for safety.
- On **macOS**:
  - If `macism` is available, enables **sis** for input-method switching and colored cursors.
  - Adds **Apple Music** helpers (play/pause, next/previous, pick playlist, show info), including a **Hydra** and Meow leader alias `<leader> M`.

*** Module map (where things live)
| Module file | Role |
|-------------+------|
| =user.el=   | Personal identity, fonts, directory layout, Org sync, macOS IME (sis), Apple Music Hydra & Meow leader glue |

*** How it works (flow)
1. **Startup**
   - Identity, fonts, switches, and directory roots are defined. Missing dirs are created.
   - If Org loads later, `org-directory` is set to your cloud path; `org-agenda-files` is (re)computed.
2. **macOS only**
   - If `macism` exists in `PATH`, **sis** config wires ABC ⇄ system IME switching and cursor coloring.
   - Apple Music helpers wrap `osascript` (sync/async) and expose a **Hydra**.
3. **Keybindings**
   - When **Hydra** is present, the Apple Music Hydra is on `C-c M`.
   - If **Meow** and **Hydra** are both loaded, `<leader> M` launches the same Hydra.

*** Troubleshooting
- **IME doesn’t toggle (macOS)** → Ensure `macism` is installed and visible in PATH; `sis` is only enabled when `(executable-find "macism")` succeeds.
- **Hydra/leader key not found** → The Apple Music Hydra needs **hydra**; the leader binding needs **meow**. Both are guarded with `with-eval-after-load`.

- **Agenda empty** → Confirm your cloud `org/` contains `.org` files and none are buried under an `archives/` path.

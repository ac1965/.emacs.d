* Configuration
** Eary-init.el
/Here's a summary of what it does:/

- *Garbage Collection Tuning*
  - You've set up a high initial threshold for garbage collection to avoid frequent pauses during startup, with a reset mechanism after initialization.

- *Native Compilation*
  - Your configuration ensures proper handling of native compilation caches and suppresses warnings.

- *File Handling*
  - Temporarily disabling `file-name-handler-alist` during startup improves loading speed.

- *GUI and Startup Settings*
  - You’ve configured a minimalist and efficient GUI setup, particularly with elements like frame resizing, menu bar handling, and fullscreen mode.

** Personal Configuration aka. tjy.el
/Here's a summary of what it does:/

1. *Personal Information & General Config*
   - Sets your full name, email, preferred font (`HackGen35`), and font size.
   - Defines important directories such as cloud storage and blog paths.
   - Adds custom elisp directories to `load-path`.
   - Sets up font configurations for Emacs when running as a daemon or in window mode.

2. *MEW Email Client*
   - Configures MEW as the email client within Emacs, with support for sending and composing emails.

3. *Org-mode Setup*
   - Configures Org-mode with custom keybindings for agenda (`C-c a`), capture (`C-c c`), and showing specific org files (`gtd.org`, `notes.org`, `kb.org`).
   - Integrates various org-mode extensions such as:
     - `org-superstar` for pretty bullets.
     - `org-journal` for daily entries.
     - `org-cliplink` for capturing web links into org files.
     - `org-download` for adding images.
     - `org-web-tools` for capturing web content.
     - `toc-org` for generating tables of contents in org and markdown files.
   - Enables `ox-hugo` for Hugo static site generation and sets up capture templates for blog posts.

4. *Org-roam Setup*
   - Sets up Org-roam for personal knowledge management and daily notes.
   - Configures keybindings for Org-roam-related tasks like inserting and capturing nodes.

5. *ChatGPT, CodeGPT, DALL-E Integration*
   - Prepares Emacs for ChatGPT, CodeGPT, and DALL-E integration using the `openai` packages.

6. *Miscellaneous Functions*
   - Adds a function to generate and insert a YouTube iframe link in org files.
   - Integrates Visual Studio Code for opening files in VSCode (`C-c C-v`).
   - Provides a function to display Emacs build information.
   - Adds a function to convert an Org-mode task to a reminder (using macOS Reminders).

*** Leaf
/This Emacs configuration sets up package management and defines essential packages using `leaf` for a more structured approach. Here's a summary/

1. *Package Management*
- Sets the package directory with `package-user-dir` using `my:d:cache`.
- Configures package archives (GNU and MELPA) and initializes the package system with `package-initialize`.

2. *Leaf Setup*
- Installs and initializes `leaf`, a package that simplifies package configuration in Emacs.
- Adds support for additional `leaf` keywords using `leaf-keywords`.

3. *Essential Packages*
- `dash`: Provides modern list manipulation functions, independent of the `cl` package.
- `diminish`: Hides minor mode names from the mode line to keep it clean.
- `leaf-convert`: Converts traditional Emacs Lisp configurations to the `leaf` format.
- `leaf-tree`: Adds an interactive sidebar for managing `leaf`-based configurations.

4. *Other Packages*
- `transient-dwim`: Binds `M-=` to dispatch context-sensitive commands using `transient-dwim-dispatch`.

This setup optimizes Emacs for clearer organization, reducing clutter in the mode line, and providing a more interactive configuration experience.

                                (cadr next-win-edges)))))
                 (splitter
                  (if (= (car this-win-edges)
                         (car (window-edges (next-window))))
                      'split-window-horizontally
                    'split-window-vertically)))
            (delete-other-windows)
            (let ((first-win (selected-window)))
              (funcall splitter)
              (if this-win-2nd (other-window 1))
              (set-window-buffer (selected-window) this-win-buffer)
              (set-window-buffer (next-window) next-win-buffer)
              (select-window first-win)
              (if this-win-2nd (other-window 1))))))

    (defun my/dired-view-file-other-window ()
      (interactive)
      (let ((file (dired-get-file-for-visit)))
        (if (file-directory-p file)
            (or (and (cdr dired-subdir-alist)
                     (dired-goto-subdir file))
                (dired file))
          (view-file-other-window file))))

    ;; Remove duplicate entries from kill-ring
    (defun my/no-kill-new-duplicate (yank)
      (setq kill-ring (delete yank kill-ring)))
    (advice-add 'kill-new :before #'my/no-kill-new-duplicate)

    ;; Clean up whitespace before saving
    (add-hook 'before-save-hook 'delete-trailing-whitespace)

/This configuration script sets up a sophisticated **completion framework** for Emacs, utilizing a combination of packages to provide fast, flexible, and user-friendly completion across various contexts. Here's an overview of the key components/

1. *Corfu (COmpletion in Region FUnction)*
   - Provides an in-buffer completion UI.
   - Enabled globally with `global-corfu-mode` and `corfu-popupinfo-mode`.
   - Configured to display completions automatically with no delay (`corfu-auto-delay . 0`) and after typing just one character (`corfu-auto-prefix . 1`).
   - Adds a manual separator insertion function (`C-s` for `corfu-insert-separator`).

2. *Vertico (VERTical Interactive COmpletion)*
   - Provides a vertical completion interface in the minibuffer.
   - Automatically enabled after Emacs initializes (`vertico-mode`).

3. *Marginalia*
   - Enhances minibuffer completion commands by annotating results with additional context (e.g., file size, modification dates).
   - Enabled globally.

4. *Embark*
   - An action framework allowing you to perform context-sensitive actions (like running commands on completion results).
   - Bound to `s-/` for invoking actions and `s-;` for performing actions in context (`embark-dwim`).
   - Adds a `consult` integration (`embark-consult`), previewing at point in embark collect mode.

5. *Affe (Asynchronous Fuzzy Finder for Emacs)*
   - A fuzzy finder for searching through files and running `grep` asynchronously.
   - Highlights matches using `orderless`, with bindings for `affe-grep` (`M-g r`) and `affe-find` (`M-g s`).

6. *Orderless*
   - A powerful completion style that matches input patterns in any order.
   - Configured to be the default completion style with adjustments for partial file name completion.

7. *Cape (Completion At Point Extensions)*
   - Enhances `completion-at-point` functions, adding support for completing file paths (`cape-file`).

8. *Consult*
   - Provides enhanced commands for navigating and searching within buffers, projects, and the wider Emacs environment.
   - Includes custom keybindings like:
     - `C-s` remapped to `consult-line` for buffer searching.
     - `M-g f` for flymake integration.
     - `consult-buffer` bound to `C-x b` for buffer switching.
     - `consult-project-buffer` bound to `C-x p b` for switching between project buffers.
     - `consult-goto-line` replaces `goto-line` for line navigation.
     - `consult-ripgrep` for project-wide searching bound to `C-M-s r`.

9. *Custom Functions*
   - `c/consult-line`: A wrapper around `consult-line` that uses the symbol at point if a `C-u` prefix is supplied.

10. *Completion History*
   - Adds a `consult-history` function to the minibuffer, allowing users to cycle through previous inputs with `C-r`.

This configuration optimizes completion for speed and accuracy, leveraging both fuzzy searching and flexible pattern matching. With utilities like `Embark`, `Consult`, and `Affe`, users can search, act on, and navigate within their files with ease.
